Overview/Problem Statement

The objective was to improve the usability and accessibility of the flight booking experience on eDreams.net. Based on user testing insights.

The goal was to redesign critical UI flows (such as on-page search, feedback/help access, and visual indicators) to improve user clarity, reduce cognitive load, and increase successful task completion rates.

High Fidelity Flight Search Redesign

The work draws from best practices seen in modern travel platforms like Google Flights, Skyscanner, and Expedia, where immediate feedback, clear UI hierarchy, and minimal page transitions are emphasized.

The redesign was also inspired by usability principles from Jakob Nielsenâ€™s heuristics, particularly visibility of system status, user control, and consistency.

Process / Solution

Technologies Used:

The redesign process included:

User Research: Conducted moderated user testing with 3 participants performing key tasks while verbalizing their thoughts. Results were documented and synthesized to highlight friction points.

Wireframing in Figma: Early grayscale wireframes tested revised flows and element hierarchy.

Design Updates Included:

  • Replacing the old homepage hotel banner with a persistent, on-page flight search interface, reducing the need for page transitions.
  • Adding loading and progress indicators for search/filter operations.
  • Increasing font size and color contrast of the Help button and making it a sticky element, improving visibility across scroll states.
  • Enhancing overall layout structure for improved readability and minimal scroll fatigue.

Results:

The final prototype resolved the major friction areas discovered in user testing:

Search initiation became faster and more intuitive due to the embedded search interface.

Help and Feedback access significantly improved with the sticky positioning and visual emphasis.

Visual feedback additions ensured users were no longer left uncertain during processing.

This work was completed on time and aligns with the deliverables defined in the project brief.
